## Key Engineering Finding: EF Core + Aggregate Roots with Private Collections

### Problem
When `CartItem.Create()` set `Id = Guid.NewGuid()`, any EF operation that triggered relationship fixup (including `DbSet.Local` access) would encounter the new entity with a **non-default Guid key**. EF Core's convention for `ValueGeneratedOnAdd` Guid keys treats a non-default value as "already persisted" → tracks the new entity as **Unchanged** instead of **Added** → `SaveChanges` issues `UPDATE` against a non-existent row → `DbUpdateConcurrencyException`.

### Root Cause Chain
1. `cart.AddItem()` creates a `CartItem` with `Id = Guid.NewGuid()` and adds to `_items`
2. Any EF context access (including `Local`, `Entry`, `IsModified`) triggers relationship fixup for the Cart entity
3. Fixup finds the new CartItem in `_items`, sees a non-default Guid → tracks as Unchanged
4. `SaveChanges` generates `UPDATE CartItems WHERE Id = X` → 0 rows → exception

### Fix
Remove `Id = Guid.NewGuid()` from `CartItem.Create()`. `CartItem.Id` stays as `Guid.Empty` until EF assigns a generated value when first tracking the entity as Added. With a default key, EF correctly identifies the entity as new.

### Supporting Fix
Configure `Cart.Items` and `Order.Items` navigations with `UsePropertyAccessMode(PropertyAccessMode.Field)`:
- These properties return `_items.AsReadOnly()` (an immutable wrapper)
- With `PreferFieldDuringConstruction` (EF default), EF may try to write to the immutable property during inverse fixup
- `PropertyAccessMode.Field` forces EF to use `_items` directly for all operations

### CartRepository.Update Pattern
To handle the full lifecycle without relying on EF's automatic collection-change detection:
1. Read `DbSet<CartItem>.Local` (no DetectChanges) for the current cart to get persisted IDs
2. Mark items in Local but not in `cart.Items` as Deleted
3. Explicitly `Add` items in `cart.Items` but not in Local
4. Skip explicit Cart scalar marking — SaveChanges' internal DetectChanges detects `UpdatedAt` via snapshot

## Decisions Made

| Decision | Detail |
|---|---|
| Cart identity | Own GUID, optional CustomerId for future auth |
| Live prices in cart | Cart carries ProductId + live price at mapping time; no price snapshot until checkout |
| Order snapshot | OrderItem captures Sku, ProductName, UnitPrice — no FK to Product (intentional) |
| Atomic checkout | Cart.Clear() + Order created in single SaveChanges call |
| Events dispatched post-save | `DispatchAsync` called after `SaveChanges` — events only fire on successful transaction |
| `DbSet.Local` over `ChangeTracker.Entries<T>()` | `Entries<T>()` triggers DetectChanges (harmful side effect); Local does not |
