## Scope

Add Catalog management with many-to-many product relationships, input validation via FluentValidation endpoint filters, and a global exception handling middleware.

---

## What Was Done

### Domain Layer
- `Catalog.UpdateDetails(name, description)`: mutation method following same pattern as `Product.UpdateDetails`
- `ICatalogRepository`: full interface — CRUD + 6 catalog-product relationship methods (`AddProductToCatalog`, `AddProductsToCatalog`, `RemoveProductFromCatalog`, `IsProductInCatalog`, `GetProductsInCatalog`, `GetProductsInCatalogPaged`)

### Infrastructure Layer
- `CatalogRepository`: full implementation
  - `GetByIdWithProductsAsync`: uses `Include(c => c.CatalogProducts).ThenInclude(cp => cp.Product)` for eager loading
  - `GetProductsInCatalogPagedAsync`: queries through junction table with `OrderBy(cp => cp.Product.Name)`
  - `AddProductsToCatalogAsync`: bulk insert via `AddRangeAsync`

### Contracts Layer
- `CatalogDto`: added `ProductCount` field
- `CatalogDetailDto`: new — includes embedded `IReadOnlyList<ProductDto> Products`
- `UpdateCatalogRequest`: new
- `AddProductsToCatalogRequest`: new — `IReadOnlyList<Guid> ProductIds`

### Api Layer
- `CatalogMappingExtensions`: `ToDto()`, `ToDto(int productCount)` overload, `ToDetailDto()`, `ToDtos()`, `ToEntity()`
- `CatalogEndpoints`: 8 endpoints — CRUD + `/products` nested routes
  - `POST /{id}/products`: validates all product IDs exist, silently skips already-added duplicates, returns count added
  - `GET /{id}`: returns `CatalogDetailDto` with embedded products via eager load
- `ValidationFilter<T>`: generic `IEndpointFilter` — resolves `IValidator<T>` from DI, returns `400 VALIDATION_FAILED` with per-field error dictionary
- Validators: `CreateProductRequest`, `UpdateProductRequest`, `CreateCatalogRequest`, `UpdateCatalogRequest`, `AddProductsToCatalogRequest`
  - Applied via `.AddEndpointFilter<ValidationFilter<T>>()` on POST/PUT endpoints
- `ExceptionHandlingMiddleware`: registered first in pipeline
  - `ArgumentException` → 400 `VALIDATION_ERROR`
  - `InvalidOperationException` → 400 `INVALID_OPERATION`
  - All others → 500 `INTERNAL_ERROR` (message sanitised, no stack trace)
- `Program.cs`: `AddValidatorsFromAssemblyContaining<CreateProductRequestValidator>()`, `UseMiddleware<ExceptionHandlingMiddleware>()`, `AddScoped<ICatalogRepository, CatalogRepository>()`, `MapCatalogEndpoints()`

## Decisions Made

| Decision | Detail |
|---|---|
| `ValidationFilter<T>` over MVC filter | Minimal API–compatible; `IEndpointFilter` is the correct abstraction for endpoint-level cross-cutting |
| Duplicate products silently skipped | Idempotent bulk-add; caller gets back count of *newly* added, not an error — consistent with REST PATCH semantics |
| `ExceptionHandlingMiddleware` before Swagger | Ensures all unhandled exceptions — including from endpoints — are caught; Swagger middleware doesn't swallow exceptions |
| `ToDto(int productCount)` overload | `GetPagedAsync` returns catalogs without `CatalogProducts` loaded; avoids N+1 by accepting explicit count (0 for MVP list view) |
| `GetByIdWithProductsAsync` separate method | Eager loading only when needed; keeps `GetByIdAsync` lightweight for update/delete paths |
