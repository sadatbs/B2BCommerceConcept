## What Was Done

### Installation
- Installed **.NET 9.0.312 SDK** via `winget install Microsoft.DotNet.SDK.9` (was not present on machine)
- Installed **dotnet-ef 9.0.4** global tool

### Solution Structure
Created 4-project solution via `dotnet new` + `dotnet sln add`:
- `src/B2B.Commerce.Domain` — class library, no external deps
- `src/B2B.Commerce.Infrastructure` — class library, references Domain
- `src/B2B.Commerce.Api` — ASP.NET Core Web API, references Domain + Infrastructure
- `tests/B2B.Commerce.Tests` — xUnit project, references Domain

### Domain Layer
- `Entities/Product.cs` — factory method `Create(...)`, `UpdatePrice`, `UpdateDetails`
- `Entities/Catalog.cs` — factory method `Create(...)`
- `Entities/CatalogProduct.cs` — junction entity with `AddedAt` metadata
- `Interfaces/IProductRepository.cs` — 7-method repository contract

### Infrastructure Layer
- `Data/CommerceDbContext.cs` — EF Core DbContext with Fluent API config (unique SKU index, precision on Price, composite PK on CatalogProduct)
- `Data/CommerceDbContextFactory.cs` — `IDesignTimeDbContextFactory` for `dotnet ef` CLI
- `Migrations/20260326092040_InitialCreate` — initial schema migration

### Tests (TDD)
5 unit tests in `tests/B2B.Commerce.Tests/Domain/ProductTests.cs`, all passing:
- `Create_WithValidData_ShouldCreateProduct`
- `Create_WithEmptySku_ShouldThrowArgumentException`
- `Create_WithNegativePrice_ShouldThrowArgumentException`
- `Create_ShouldNormalizeSku_ToUpperCase`
- `UpdatePrice_WithValidPrice_ShouldUpdatePriceAndTimestamp`

### Docker
- `docker-compose.yml` with `postgres:16-alpine`, named volume `pgdata`, healthcheck via `pg_isready`
- Container `b2b-db` running and healthy; migration applied successfully

## Decisions Made

| Decision | Detail |
|---|---|
| NuGet version pinning | Latest EF Core / Npgsql packages resolve to .NET 10 targets; pinned all to `9.0.4` |
| `Microsoft.EntityFrameworkCore` added to Api | Webapi template pulled EF Core 9.0.1 transitively; explicit `9.0.4` reference eliminated MSB3277 warning |
| Design-time factory in Infrastructure | Allows `dotnet ef` to run without the Api being wired up |
| `dotnet ef --startup-project` set to Infrastructure | Infrastructure owns the DbContext and the factory |
