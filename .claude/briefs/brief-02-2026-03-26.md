## Scope

Add the Contracts project (DTOs/requests), implement `ProductRepository`, create 7 Minimal API endpoints, and cover everything with repository unit tests and integration tests via TestContainers.

---

## What Was Done

### New Project: B2B.Commerce.Contracts
- Zero external dependencies — pure C# records
- `Products/`: `ProductDto`, `CreateProductRequest`, `UpdateProductRequest`, `UpdateProductPriceRequest`
- `Catalogs/`: `CatalogDto`, `CreateCatalogRequest` (stub, used in Brief #03)
- `Common/`: `PagedRequest` (with computed `Skip`), `PagedResponse<T>` (with `TotalPages`, `HasNextPage`, `HasPreviousPage`), `ErrorResponse` (with static factory methods: `NotFound`, `Conflict`, `ValidationFailed`)
- Referenced by Api and Tests projects

### Domain Layer Update
- `IProductRepository`: added `GetPagedAsync(int skip, int take)` returning `(IReadOnlyList<Product> Items, int TotalCount)` tuple

### Infrastructure Layer
- `Repositories/ProductRepository.cs`: full `IProductRepository` implementation
  - SKU normalization in `GetBySkuAsync` and `SkuExistsAsync` (matches domain behavior)
  - `GetPagedAsync` uses `CountAsync` + `OrderBy(Name).Skip.Take`

### Api Layer
- `Mapping/ProductMappingExtensions.cs`: static extension methods (`ToDto`, `ToDtos`, `ToEntity`)
  - Removed nullable overload of `ToDto` — caused CS0111 (same parameter types as non-nullable overload)
- `Endpoints/ProductEndpoints.cs`: 7 Minimal API endpoints using `TypedResults` and `Results<T1, T2>` union types
- `Program.cs`: replaced template stub with full DI registration, Swagger, endpoint mapping, `public partial class Program` for test factory
- `appsettings.Development.json`: added `ConnectionStrings:DefaultConnection`

**Total: 22 tests, all passing** (5 domain + 8 repository + 9 integration)

## Decisions Made

| Decision | Detail |
|---|---|
| Contracts has zero deps | Brief spec requirement; keeps the contract layer deployable anywhere |
| Tuple return on `GetPagedAsync` | Avoids creating a wrapper type for a simple pair of values; acceptable at this scale |
| `TypedResults` over `Results` | Enables compile-time response type checking and accurate OpenAPI schema generation |
| `public partial class Program` | Required for `WebApplicationFactory<Program>` in integration tests — standard ASP.NET pattern |
| Swashbuckle over `AddOpenApi` | `AddOpenApi` (Microsoft's built-in) doesn't include Swagger UI; Swashbuckle provides both |
| `Microsoft.EntityFrameworkCore.Relational 9.0.4` pinned in Api | Swashbuckle pulled EF Relational 9.0.1 transitively; explicit pin resolved MSB3277 warning |
| `PostgreSqlBuilder("postgres:16-alpine")` | Parameterless constructor is deprecated in current TestContainers version |
| Per-test container isolation | `IntegrationTestBase : IAsyncLifetime` — each test class gets its own container and migrated DB |
